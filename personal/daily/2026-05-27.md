## 学到了什么 · AI 基础概念卡片

### 1. Agent
智能体 / 代理。

### 2. LLM（大语言模型）
输入 → LLM → 输出。**LLM 本身不能调用工具**，只能输出"我想用这个工具"的文字，真正执行的是外部平台 / agent loop。

### 3. Tokenizer / Token / Token ID
- **Tokenizer（分词器）**：负责把文字编码成 token，再把 token 解码回文字。
- **Token**：模型处理文本的最小单元（可能是一个词、子词、甚至一个字符），不一定等于一个汉字 / 单词。
- **Token ID**：token 对应的数字编号。模型内部其实只认数字。

### 4. Context（上下文）& Context Window
- **Context**：一次调用塞给 LLM 的全部信息——对话历史、system prompt、用户当前问题、工具列表等。
- ⚠️ LLM **本身没有记忆**——每次调用都要把历史重新拼进 context。所谓"记忆"是平台帮我们拼好再发给模型。
- **Context Window（上下文窗口）**：一次能容纳的最大 token 数。

### 5. Prompt（提示词）
给 LLM 的具体指令 / 问题。**直接决定输出质量**。常分两种：
- **System prompt**：定义角色、风格、规则（隐藏在背后）。
- **User prompt**：用户当下的问题。

### 6. Tool（工具）
给 LLM 提供的外部能力接口，让它能"感知 + 影响"外部世界。分工：
- LLM：**选哪个工具 + 传什么参数 + 看完结果再总结**。
- 平台 / agent loop：**真正去执行**这个工具调用。

### 7. MCP（Model Context Protocol）
统一的工具 / 资源接入规范。让同一套工具能被不同 LLM 客户端复用，避免每接一个模型重写一遍。

### 8. Agent（智能体）
能**自主规划 + 自主调用工具**的系统。五大组成：
1. **LLM**：推理大脑。
2. **Prompt（system prompt）**：角色 + 规则。
3. **Memory（记忆）**：跨轮次保留的信息。
4. **External Knowledge（外部知识 / RAG）**：临时检索的资料。
5. **Tools（工具）**：可调用的外部能力。

### 9. ReAct 模式
**Reasoning + Acting（推理 + 行动）**。对应 **agent loop**：

> 思考 → 行动（调工具）→ 观察结果 → 再思考 …… 直到完成目标。

### 10. Agent Skill
本质是一份 **Markdown 说明文档**，两层结构：
- **元数据层**：`name`（叫什么）+ `description`（负责做什么——给 LLM 自己看的"什么时候该用我"，写得越具体越容易被正确调用）。
- **指令层**：具体怎么做、有哪些步骤、注意事项。

### 11. Workflow（工作流）
**人工设计的固定流程**，AI 按既定路线走，不做自主决策。

### 12. Agent vs Workflow（怎么判断）
关键差异：**路径是人定的，还是 AI 自己选的**？
- 人定路径 → **Workflow**。
- AI 自主规划、会思考会推理、不满意会自我迭代 loop → **Agent**。

判断一个产品是不是 agent：看它**能不能围绕目标自主完成工作 + 自检 + 继续迭代**。
