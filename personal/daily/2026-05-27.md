## 学到了什么 · AI 基础概念卡片

### 1. LLM（大语言模型）
**是什么**：输入文字 → LLM → 输出文字。本质是"超大号的下一个 token 预测器"。
**例子**：GPT-4、Claude、通义千问、DeepSeek。
**⚠️ 误区 / 边界**：LLM **不能自己调用工具 / 执行代码 / 联网**——它只能**输出"我想用这个工具"的文字**，真正执行的是外面的平台（agent loop）。看到 ChatGPT 联网，那是 OpenAI 在 LLM 外面套了一层执行器，不是 LLM 自己长出来的能力。

### 2. Tokenizer / Token / Token ID
**是什么**：
- **Tokenizer（分词器）**：把文字切碎、编码成数字；输出时再把数字解码回文字。
- **Token**：模型处理文本的最小单元，可能是一个词、一个子词、甚至一个字符。
- **Token ID**：token 对应的数字编号——模型内部其实只认数字。
**例子**：英文 "hello" ≈ 1 token；中文"你好"可能被切成 2 个 token，但一个生僻字可能要 2–3 个 token 才能表示。
**⚠️ 误区**：很多人以为"一个汉字 = 1 token"——错。中文通常 **1 汉字 ≈ 1.5–2 tokens**，所以同一个 context window 装中文比英文吃亏。

### 3. Context（上下文）& Context Window
**是什么**：
- **Context** = 一次调用塞给 LLM 的**全部信息**：对话历史 + system prompt + 当前问题 + 工具列表。
- **Context Window** = 一次最多能塞多少 token（GPT-4o 约 128k，Claude Opus 4 可达 1M）。
**例子**：你和 ChatGPT 聊了 20 轮，每问一次新问题，前 20 轮历史**都要重新塞进去一次**才能让它"记得"。
**⚠️ 误区**：LLM **本身没有记忆**。看上去"记得我们昨天聊过"——其实是平台帮你把历史拼好再发给模型。所谓"长记忆"= 把更多历史拼进 context（或者用外部数据库存起来需要时检索）。

### 4. Prompt（提示词）
**是什么**：发给 LLM 的具体指令 / 问题。**直接决定输出质量**。
- **System prompt**：定义角色、风格、规则（在背后，用户通常看不见）。
- **User prompt**：用户当下问的问题。
**例子**：
- 烂 prompt：「帮我写个文案」
- 好 prompt：「你是一个 Web3 新手向的科普写作者，请用 200 字解释什么是 MCP，要包含一个生活化的类比，不要用术语」
**⚠️ 边界**：Prompt 不是越长越好——啰嗦反而稀释重点。原则：**角色 + 任务 + 输出格式 + 约束**四件事讲清就行。

### 5. Tool（工具）
**是什么**：给 LLM 接外部能力的接口——搜网页、查数据库、发邮件、调链上数据都算。
**分工**：
- LLM：**选哪个工具、传什么参数、看到结果后做总结**。
- 平台 / agent loop：**真正去执行**这次调用。
**例子**：用户问"今天上海天气"→ LLM 输出「我要调 `get_weather(city="Shanghai")`」→ 平台真的去调 API → 把"22 度"塞回 context → LLM 再用自然语言回答。
**⚠️ 误区**：以为"接了工具的 LLM 就成了 Agent"——不对，**手动一问一答触发工具**只是 tool use，不是 agent。Agent 还需要自己决定**调几次 / 调完看不看结果 / 要不要换工具**。

### 6. MCP（Model Context Protocol）
**是什么**：Anthropic 提出的**统一工具接入协议**——把"工具 / 资源"标准化成 server，谁都能接。
**例子**：写好一个 GitHub MCP server，Claude Desktop、Cursor、其他支持 MCP 的客户端**都能直接用**，不用每个平台重新写一遍集成。
**⚠️ 边界**：MCP 是**协议层**，不是模型也不是 agent 框架。它解决的是"工具怎么接"的标准化问题，不解决"模型怎么思考"。类比：MCP : LLM ≈ USB-C : 笔记本——统一接口而已。

### 7. Agent（智能体）
**是什么**：能**自主规划 + 自主调用工具**的系统。五大组成：
1. **LLM**：推理大脑。
2. **Prompt（system prompt）**：角色 + 行为规则。
3. **Memory（记忆）**：跨轮次保留的信息（比如「我之前帮你 fork 过 repo」）。
4. **External Knowledge（外部知识 / RAG）**：临时检索的资料（比如查 Handbook）。
5. **Tools（工具）**：可调用的外部能力。
**例子**：AutoGPT、Devin、Claude Code、Manus，以及今天我用的 AI × Web3 School Learning Agent。
**⚠️ 边界**：Agent ≠ ChatBot。一句话差别：**ChatBot 等你下一句，Agent 等任务完成**。

### 8. ReAct 模式
**是什么**：**Reasoning + Acting（推理 + 行动）**——agent 的标准工作循环：

> 思考 → 行动（调工具）→ 观察结果 → 再思考 …… 直到完成目标，或主动停下。

**例子**（今天用 Learning Agent 整理这份笔记）：
1. **思考**：用户给了零散概念，我应该先找出错误点
2. **行动**：读 memory、读模板、读昨天的 daily note
3. **观察**：发现项目用 fork 模式，且 daily 走 `personal/daily/`
4. **再思考**：那我应该按模板写到 `personal/daily/2026-05-27.md`
5. **再行动**：写文件 → 展示 git status → 等用户确认 commit

**⚠️ 误区**：以为 ReAct 一定要写代码——不一定，**人脑也可以跑 ReAct**：思考一下、做一步、看结果、再决定下一步。

### 9. Agent Skill
**是什么**：本质是一份 **Markdown 说明文档**，两层结构：
- **元数据层**：`name`（叫什么）+ `description`（"什么时候应该调用我"——给 LLM 自己读的）。
- **指令层**：具体怎么做、步骤、注意事项。
**例子**：Claude Code 的 `verify` skill——description 写明"用来跑应用并验证某个改动是否真的生效"，所以当用户说"帮我验证一下这个 PR"，Claude 就会自动想到要用它。
**⚠️ 关键**：description 写得越具体，被正确调用的概率越高。写得太泛（比如"处理代码相关任务"），LLM 反而每次都犹豫该不该用。

### 10. Workflow（工作流）
**是什么**：**人工设计的固定流程**——AI 按预设路线走，不做自主决策。
**例子**：n8n / Zapier / Coze / Dify 里拖出来的流程图：「收到邮件 → 提取关键词 → 调 LLM 总结 → 发到飞书」。每一步都是人定好的。
**⚠️ 边界**：Workflow 适合**流程确定 + 重复执行**的场景（自动化报表、数据清洗）；不适合**目标明确但路径不确定**的场景（"帮我研究下 XX 赛道"）——那种该用 agent。

### 11. Agent vs Workflow（怎么判断）
**核心差异**：**路径是人定的，还是 AI 自己选的**？
- 人定路径 → **Workflow**
- AI 自主规划 + 会思考会推理 + 不满意会自我迭代 loop → **Agent**

**判断一个产品是不是 agent**：看它**能不能围绕目标自主完成工作 + 自检 + 继续迭代**。
- ChatGPT 默认对话：偏 chat，不是 agent
- ChatGPT 加上"Deep Research"模式：是 agent（自主多轮搜索 + 综合）
- Zapier 的固定 flow：workflow
- Devin / Claude Code：agent

## AI 辅助说明（怎么复核 / 怎么改写）

今天这份笔记我用了 Claude 协助，但不是直接复制 AI 输出：

1. **我先用自己的话**把 12 个概念草草列了一遍（编号还乱了——跳了 8、重复了 11）。
2. **让 Claude 纠错**：它指出 3 个我记错的地方——「token = 文字」（不准，应该是"文本最小单元"）、「context 是 LLM 临时记忆」（错，LLM 没有记忆，是平台帮拼的）、「MCP 是统一接入规范」（不够准确，应该是 Model Context Protocol，工具协议层）。
3. **我逐条复核**：每个纠错点我都问"为什么"——比如 token 那条，我现场去 OpenAI tokenizer demo（platform.openai.com/tokenizer）数了一下"你好"的 token 数，验证确实不等于汉字数。
4. **例子和误区都尽量用自己理解的版本**写——比如"Agent vs ChatBot = 等下一句 vs 等任务完成"这一句是我自己的总结，不是 AI 给的。
5. **AI 给的内容里如果我自己都讲不清的，就删掉**——宁可少一条，不要塞看上去高级但自己说不出口的术语。

> 原则：AI 当**编辑 + 纠错员**，不当**代笔**。如果一段话我读完没法用自己的话复述一遍，那就不算我学会了。
